# `ios`ディレクトリのドキュメント

このドキュメントは`ios`ディレクトリ内の主要なファイルとその役割の概要をまとめたものです。

## 主要なファイルとディレクトリ

*   `Runner.xcworkspace`: Xcodeでプロジェクトを開く際に使用するファイルです。`Runner.xcodeproj`に加えて、依存関係（CocoaPodsなど）を管理します。
*   `Runner.xcodeproj`: Xcodeプロジェクトファイルです。アプリのターゲット設定、ビルド設定、ファイル構成などが含まれています。
*   `Runner/`: アプリケーションのソースコードやリソースを格納するメインディレクトリです。
*   `Runner/Info.plist`: アプリケーションの重要な設定情報を含むプロパティリストファイルです。バンドルID、表示名、バージョン番号、URLスキーム、プライバシー関連の権限設定などが定義されます。
*   `Runner/AppDelegate.swift`: アプリケーションのライフサイクルイベント（起動、終了、バックグラウンドへの移行など）を処理するためのデリゲートクラスです。FlutterとネイティブiOSコードの連携や、プッシュ通知などのプラグイン初期化を行います。
*   `GoogleService-Info.plist`: FirebaseプロジェクトとiOSアプリを連携させるための設定ファイルです。APIキーやデータベースURL、GoogleサインインのクライアントIDなどが含まれています。
*   `Runner/Assets.xcassets`: アプリアイコンや起動画像などの画像リソースを管理します。
*   `Flutter/`: Flutterフレームワーク関連のファイルが含まれます。`Debug.xcconfig`と`Release.xcconfig`は、ビルド構成ごとの変数を定義します。
